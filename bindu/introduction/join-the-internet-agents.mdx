## 🚀 The Fast Way - Bindu Installer

Time to first agent: **~2 minutes** ⏱️

This is the fastest way to get started. Super simple!

### Step 1: Install Bindu 📦

```bash
curl -fsSL https://getbindu.com/install-bindu.sh | bash
```

### Step 2: Create Your Agent 🤖

Navigate to where you want your agent project and run:

```bash
bindu create-agent
```

<Tip>
💡 **Pro Tip**: The Bindu installer is way better than cookiecutter - it's designed specifically for agents!
</Tip>

<Note>
For detailed documentation, see the [Template Overview](/bindu/create-bindu-agent/overview).
</Note>

## 🎯 Already Built Something?

You've got an agent running? Awesome! 🎉  
Maybe it's in LangChain. Maybe CrewAI. Maybe custom code.

Doesn't matter what framework you used.

You can **Bindu-fy it in just 5 minutes**! ⚡

Two things you need: a config file and the `bindufy` wrapper.  
That's it. Seriously! 🤓

### 📝 Step 1: Create `agent_config.json`

```json
{
  "author": "your.email@example.com",
  "name": "my_existing_agent",
  "description": "My agent with Bindu superpowers",
  "version": "1.0.0",
  "deployment": {
    "url": "http://localhost:3773",
    "expose": true
  }
}
```

<Note>
📚 **Want More Options?** Check the [Configuration Reference](/bindu/create-bindu-agent/configuration) for A2A, AP2, and X402 protocol settings!
</Note>

### 🧩 Step 2: Wrap Your Agent with `bindufy`

```python
from bindu.penguin.bindufy import bindufy
from agno.agent import Agent
from agno.models.openai import OpenAIChat
import json

# Load your config
with open("agent_config.json", "r") as f:
    config = json.load(f)

# Your existing agent
my_agent = Agent(
    instructions="Your agent instructions",
    model=OpenAIChat(id="gpt-4o"),
)

# Handler function
def agent_handler(messages: list[dict[str, str]]):
    result = my_agent.run(input=messages)
    return result

# Bindufy it! 🎉
bindufy(my_agent, config, agent_handler)
```

**And... you're DONE!** 🎊

Your agent is now live and ready to rock the Internet of Agents! 🌐

No infrastructure setup.  
No protocol implementation.  
No weeks of DevOps work.

Just your agent, now accessible to the world.

## 🎉 What Happens During Setup?

When you run `bindu create-agent`, you'll get an awesome interactive setup! 🎮

You'll be prompted for:
- 📋 **Project details** - Name, description, author info
- 🤖 **Agent framework** - Agno, LangChain, CrewAI, etc.
- ⚡ **Skills & Capabilities** - Choose from 29+ built-in skills
- 🔐 **Authentication** - Ory Hydra OAuth2 setup
- 💾 **Storage** - Memory (ephemeral) or PostgreSQL
- ⏰ **Scheduler** - Memory or Redis-based
- 📊 **Observability** - Phoenix, Jaeger, Langfuse options
- 🛡️ **Security Features** - DID, PKI, or combined
- 🔄 **CI/CD** - GitHub Actions workflows
- 📄 **License type** - MIT, Apache, BSD, GPL, ISC

Then boom! 💥 Your agent project is ready with:

```
your-agent/
├── your_agent/
│   ├── skills/               # Template for adding agent skills
|   ├── agent_config.json     # Agent configuration with A2A/AP2/X402 settings
|   ├── main.py               # Agent entry point (Bindu-fied!)
│   └── __init__.py
├── tests/                    # Pre-configured pytest tests
├── pyproject.toml            # Dependencies managed by uv
├── Dockerfile                # Ready for containerization
├── .github/workflows/        # CI/CD pipelines
└── README.md                 # Complete setup instructions
```

## 🔥 Fire It Up!

Ready to start your agent? Let's do this! 🚀

### Step 1: Navigate to Your Agent Directory

```bash
cd your-agent
```

### Step 2: Install Dependencies (Super Fast with uv!) ⚡

```bash
uv sync
```

### Step 3: Start Your Agent 🤖

```bash
uv run python -m your_agent.main
```

## 😱 When Things Go Wrong

They usually don't. But hey, we've all been there! 🤷‍♂️

<AccordionGroup>
  <Accordion title="🔍 bindu command not found">
    You need to install the Bindu installer first. No worries!
    
    ```bash
    curl -fsSL https://getbindu.com/install-bindu.sh | bash
    ```
    
    Then restart your terminal and try again. 🔄
  </Accordion>
  
  <Accordion title="⚡ uv not found">
    The Bindu installer needs uv for dependency management.
    
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
    
    Restart your terminal and you're good to go! 🚀
  </Accordion>
  
  <Accordion title="🚫 Port already in use">
    Something's already running on port 3773. Happens to the best of us!
    
    Change the `deployment_port` in your `agent_config.json` to something else (like 3774 or 9000).
    
    Or kill whatever's using that port. Your call! 💪
  </Accordion>
  
  <Accordion title="🔐 Authentication errors">
    You probably haven't set up Auth0 credentials yet.
    
    Check the generated README for details on environment variables.
    
    Or disable auth in your config if you're just testing. We won't tell! 🤫
  </Accordion>
  
  <Accordion title="📦 Dependencies not installing">
    Sometimes the virtual environment gets corrupted. It happens!
    
    Nuke it and start fresh:
    ```bash
    rm -rf .venv
    uv sync
    ```
    
    Should work now. Like magic! ✨
  </Accordion>
</AccordionGroup>
