# Bindu Agent API

The Bindu Agent API is a comprehensive A2A Protocol compliant API supporting JSON-RPC 2.0 methods for agent communication, task management, context management, DID resolution, and payment processing.

## Base URL

```
http://localhost:8030
```

<Note>
  Default development server runs on port 8030. Production deployments should use HTTPS.
</Note>

---

## Authentication

Most endpoints support optional Bearer token authentication:

```bash
Authorization: Bearer <your_jwt_token>
```

**Public Endpoints** (no authentication required):
- `/.well-known/agent.json` - Agent discovery
- `/did/resolve` - DID resolution
- `/agent/skills` - Skills listing
- `/agent/skills/{skillId}` - Skill details

**Protected Endpoints** (authentication required):
- `/` - JSON-RPC methods
- `/api/start-payment-session` - Payment initiation
- `/api/payment-status/{sessionId}` - Payment status

## Quick Start

### Send a Message

```bash
curl -X POST http://localhost:8030/ \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "message/send",
    "params": {
      "message": {
        "role": "user",
        "parts": [
          {
            "kind": "text",
            "text": "Hello, agent!"
          }
        ],
        "kind": "message",
        "messageId": "msg-123",
        "contextId": "ctx-123",
        "taskId": "task-123"
      }
    },
    "id": "req-123"
  }'
```

### Get Agent Card

```bash
curl http://localhost:8030/.well-known/agent.json
```

### List Skills

```bash
curl http://localhost:8030/agent/skills
```

---

## JSON-RPC Methods

| Method | Description |
|--------|-------------|
| `message/send` | Send messages to agent |
| `message/stream` | Stream messages from agent |
| `tasks/get` | Get task status |
| `tasks/list` | List all tasks |
| `tasks/cancel` | Cancel a task |
| `tasks/feedback` | Submit feedback for a task |
| `contexts/list` | List conversation contexts |
| `contexts/clear` | Clear a context |

---

## Response Format

### Success Response

```json
{
  "jsonrpc": "2.0",
  "result": {
    "task": {
      "taskId": "task-123",
      "status": {
        "state": "completed",
        "timestamp": "2025-10-26T14:47:52Z"
      },
      "artifacts": [
        {
          "kind": "text",
          "text": "Response from agent"
        }
      ]
    }
  },
  "id": "req-123"
}
```

### Error Response

```json
{
  "jsonrpc": "2.0",
  "error": {
    "code": -32602,
    "message": "Invalid params",
    "data": {
      "details": "Missing required field: taskId"
    }
  },
  "id": "req-123"
}
```

---

## Error Codes

| Code | Message | Description |
|------|---------|-------------|
| `-32700` | Parse error | Invalid JSON |
| `-32600` | Invalid Request | Invalid JSON-RPC |
| `-32601` | Method not found | Unknown method |
| `-32602` | Invalid params | Invalid parameters |
| `-32603` | Internal error | Server error |
| `-32001` | Authentication required | No auth token |
| `-32002` | Invalid token | Token verification failed |
| `-32003` | Insufficient permissions | Missing permissions |

## Rate Limiting

<Warning>
  Rate limits apply to protect server resources. Implement exponential backoff for retries.
</Warning>

**Default Limits:**
- 100 requests per minute per IP
- 1000 requests per hour per IP

**Headers:**
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1635264000
```

## Versioning

The API follows semantic versioning:

**Current Version:** `1.0.0`

Version is included in the agent card:

```json
{
  "version": "1.0.0"
}
```

## SDKs and Libraries

### Python

```python
from bindu.client import BinduClient

client = BinduClient("http://localhost:8030")
response = client.send_message("Hello, agent!")
```

```typescript
import { BinduClient } from '@bindu/client';

const client = new BinduClient('http://localhost:8030');
const response = await client.sendMessage('Hello, agent!');
```
