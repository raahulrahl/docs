# Multi-Provider Authentication

## Overview

Comprehensive authentication support for multiple identity providers, enabling seamless integration with enterprise SSO and popular OAuth providers.

## Supported Providers

### AuthKit
Modern authentication infrastructure with built-in security best practices.

**Features:**
- Passwordless authentication
- Magic link login
- Multi-factor authentication (MFA)
- Session management
- User impersonation for support

### GitHub OAuth
Authenticate users with their GitHub accounts.

**Features:**
- OAuth 2.0 flow
- Organization membership verification
- Team-based access control
- Repository access scopes

### AWS Cognito
Enterprise-grade authentication from Amazon Web Services.

**Features:**
- User pools and identity pools
- Social identity providers
- SAML 2.0 federation
- Advanced security features
- Lambda triggers for custom logic

### Google OAuth
Sign in with Google for consumer and workspace accounts.

**Features:**
- OAuth 2.0 and OpenID Connect
- Google Workspace integration
- Domain verification
- Granular scopes

### Azure (Microsoft Entra)
Microsoft's enterprise identity platform.

**Features:**
- Azure Active Directory integration
- Conditional access policies
- Multi-factor authentication
- B2B and B2C scenarios
- Enterprise application integration

## Implementation

### Configuration

```python
from bindu.auth import AuthConfig, Provider

auth_config = AuthConfig(
    providers=[
        Provider.AUTHKIT(
            client_id="your-client-id",
            client_secret="your-secret",
            redirect_uri="https://your-app.com/callback"
        ),
        Provider.GITHUB(
            client_id="github-client-id",
            client_secret="github-secret",
            allowed_orgs=["your-org"]
        ),
        Provider.GOOGLE(
            client_id="google-client-id",
            client_secret="google-secret",
            hosted_domain="your-company.com"
        ),
        Provider.AZURE(
            tenant_id="your-tenant-id",
            client_id="azure-client-id",
            client_secret="azure-secret"
        ),
        Provider.AWS_COGNITO(
            region="us-east-1",
            user_pool_id="pool-id",
            client_id="cognito-client-id"
        ),
    ],
    default_provider="authkit",
    session_duration=timedelta(hours=24),
)
```

### Usage

```python
from bindu.auth import authenticate

@authenticate(providers=["github", "google"])
async def protected_endpoint(request):
    user = request.user
    # Access user information
    print(f"User {user.email} from {user.provider}")
```

## Features

### Single Sign-On (SSO)
- Seamless authentication across multiple apps
- Provider-agnostic user sessions
- Automatic token refresh

### Role Mapping
```python
# Map provider roles to Bindu trust levels
role_mapping = {
    "github:org-admin": TrustLevel.ADMIN,
    "azure:contributor": TrustLevel.OPERATOR,
    "google:viewer": TrustLevel.VIEWER,
}
```

### Multi-Factor Authentication
- TOTP (Time-based One-Time Password)
- SMS verification
- Email verification
- Biometric authentication
- Hardware security keys

### Session Management
- Secure session storage
- Automatic expiration
- Refresh token rotation
- Concurrent session limits
- Device tracking

## Security Features

- **PKCE** - Proof Key for Code Exchange for OAuth
- **State Parameter** - CSRF protection
- **Nonce** - Replay attack prevention
- **Token Encryption** - Secure token storage
- **Rate Limiting** - Brute force protection
